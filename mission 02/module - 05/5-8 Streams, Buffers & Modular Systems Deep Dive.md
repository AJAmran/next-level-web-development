Node.js-এর ভেতরে কী ঘটছে সেটা বুঝতে হলে **Streams**, **Buffers** এবং **Modular System** সম্পর্কে জানা খুব জরুরি। এগুলোই Node.js-কে এতোটা পাওয়ারফুল এবং এফিসিয়েন্ট করে তোলে।

চলো একদম ভেতর থেকে সব খুঁটিনাটি আলোচনা করি।

---

## ১. Buffer: ডাটার অস্থায়ী ওয়েটিং রুম

স্ট্রিমে যাওয়ার আগে বাফারটা বোঝা দরকার। ধরো তুমি একটা মুভি দেখছ। তোমার ইন্টারনেট স্লো হলে দেখবে লোডিং হচ্ছে। আসলে তখন ব্যাকগ্রাউন্ডে ডাটা এসে একটা জায়গায় জমা হচ্ছে। এই অস্থায়ী স্টোরেজটাই হলো **Buffer**।

* Node.js-এ **Buffer** ক্লাস ব্যবহার করা হয় বাইনারি ডাটা (Raw binary data) হ্যান্ডেল করার জন্য।
* এটি ফিক্সড সাইজ মেমরি অ্যালোকেট করে।

---

## ২. Streams: ডাটার নিরবচ্ছিন্ন প্রবাহ

পুরো একটা ফাইল একসাথে মেমরিতে লোড না করে ছোট ছোট টুকরো (Chunks) করে প্রসেস করাকেই **Streams** বলে। এটা মেমরি বাঁচায় এবং অ্যাপ্লিকেশনের স্পিড বাড়ায়।

### স্টরিমের ৪টি ধরন:

1. **Readable Stream:** শুধু ডাটা পড়া যায়।
* *উদাহরণ:* `fs.createReadStream()`, HTTP Request।


2. **Writable Stream:** ডাটা লেখা যায়।
* *উদাহরণ:* `fs.createWriteStream()`, HTTP Response।


3. **Duplex Stream:** একই সাথে পড়া এবং লেখা যায়। (পড়া এবং লেখার চ্যানেল আলাদা থাকে)।
* *উদাহরণ:* TCP Sockets।


4. **Transform Stream:** এটি এক ধরণের ডুপ্লেক্স স্ট্রিম, তবে এটি ডাটা পড়ার সময় তা পরিবর্তন (Modify) করে আউটপুট দেয়।
* *উদাহরণ:* `zlib.createGzip` (ফাইল কম্প্রেস করা)।

## ৩. Modular System: কোড গোছানোর শিল্প

Node.js-এ কোডকে ছোট ছোট ভাগে ভাগ করে রাখা হয় যাতে বারবার ব্যবহার করা যায়। একেই মডুলার সিস্টেম বলে।

### CommonJS (CJS) vs ESM (ES Modules)

আগে শুধু CommonJS ছিল, এখন আধুনিক জাভাস্ক্রিপ্টে ESM বেশি ব্যবহৃত হয়।

| ফিচার | CommonJS (CJS) | ES Modules (ESM) |
| --- | --- | --- |
| **Syntax** | `require()` / `module.exports` | `import` / `export` |
| **Default** | Node.js-এর ডিফল্ট সিস্টেম | `.mjs` ফাইল বা `package.json`-এ `"type": "module"` দিলে চলে |
| **Loading** | Synchronous (ব্লকিং) | Asynchronous (নন-ব্লকিং) |

## ৪. মডিউলের প্রকারভেদ

Node.js-এ আমরা প্রধানত ৩ ধরণের মডিউল ব্যবহার করি:

### ক) Local Modules

এগুলো হলো আমাদের নিজেদের তৈরি ফাইল।

* **ব্যবহার:** এক ফাইল থেকে অন্য ফাইলে ফাংশন বা অবজেক্ট পাঠানোর জন্য।
* *উদাহরণ:* `const myFunc = require('./myModule');`

### খ) Built-in Modules

Node.js ইনস্টল করলেই এগুলো সাথে পাওয়া যায়। আলাদা করে ডাউনলোড করতে হয় না।

* `fs`: ফাইল সিস্টেম হ্যান্ডেল করার জন্য।
* `path`: ডিরেক্টরি পাথ নিয়ে কাজ করার জন্য।
* `http`: সার্ভার তৈরি করার জন্য।
* `os`: অপারেটিং সিস্টেমের ইনফরমেশন পাওয়ার জন্য।

### গ) Third-party Modules

এগুলো অন্য ডেভেলপারদের বানানো লাইব্রেরি যা আমরা **NPM (Node Package Manager)** এর মাধ্যমে নামাই।

* **ব্যবহার:** `npm install express` বা `npm install lodash`
* এটি `node_modules` ফোল্ডারে জমা হয়।

---

### একটি প্রো-টিপ (The Pipe Method)

স্ট্রিমের ক্ষেত্রে একটি চমৎকার মেথড হলো `pipe()`। এটি একটি রিডেবল স্ট্রিমকে সরাসরি রাইটেবল স্ট্রিমের সাথে কানেক্ট করে দেয়।

```javascript
const fs = require('fs');
const readable = fs.createReadStream('file.txt');
const writable = fs.createWriteStream('copy.txt');

// ফাইল কপি করার সবচেয়ে এফিসিয়েন্ট উপায়
readable.pipe(writable);

``
